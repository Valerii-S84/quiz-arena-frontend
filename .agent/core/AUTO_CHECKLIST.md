# AUTO_CHECKLIST

Перед завершенням задачі агент зобов'язаний перевірити:

## 1. Scope
- scope задачі не розширено;
- побічні проблеми поза запитом не виправлялись;
- `helpful extras` не додані.

## 2. Межі змін
- змінено тільки потрібні файли, без жодних змін поза задачею;
- несуміжний рефакторинг не виконано;
- залежності, CI/CD, infra, prod-конфіги не змінювались
  без прямого запиту;
- нові migrations додавались тільки якщо задача класифікована
  як FEATURE MINOR або MAJOR і план підтверджений користувачем.

## 3. Доказовість
- факти відділені від висновків;
- припущення не видані за факт;
- недоведене явно позначене;
- рекомендації без прямого запиту не додані.

## 4. Перевірки
- запущено релевантні перевірки;
- якщо каталог `tests/` відсутній, `pytest tests/ -q --tb=short` не вважається обов'язковою перевіркою;
- якщо змінено лише нормативні файли, перевірка складається з reread змінених файлів, `rg`-перевірки ключових інваріантів і явної звірки на відсутність суперечностей;
- результати перевірок враховано у відповіді;
- падіння перевірок не приховано;
- якщо перевірки впали — задача не позначається як done без явного підтвердження від користувача;
- якщо перевірки не запускались, це прямо вказано;
- якщо щось не вдалося перевірити, це внесено в `Недоведено / ризики`.

## 5. Фінальний звіт
- відповідь подана у форматі з `TASK_OUTPUT_FORMAT.md`; якщо файл недоступний — використовуй формат з `AGENTS.md` розділ 14.6;
- звіт короткий і доказовий;
- довга преамбула відсутня;
- поради без запиту відсутні;
- нав'язані наступні кроки відсутні.
