# DEFINITION_OF_DONE

Задача вважається завершеною тільки якщо виконано всі умови нижче.

## 1. Done

Задача є `done` тільки якщо:

- scope задачі виконано в межах явного запиту;
- змінено тільки потрібні файли, без жодних змін поза запитом;
- побічні проблеми без прямого дозволу не виправлялись;
- заявлений результат відповідає фактичному стану репозиторію;
- релевантні перевірки виконані, або явно зафіксовано, що саме не було перевірено;
- недоведене не замасковане як підтверджений результат;
- фінальна відповідь подана у форматі з `TASK_OUTPUT_FORMAT.md`.

## 2. Обов'язкові умови завершення

Перед позначенням задачі як завершеної має бути підтверджено:

- відсутність scope drift;
- відповідність змін заявленій задачі;
- узгодженість змін між собою;
- відповідність фінального звіту фактичному стану репозиторію.

## 3. Що не є done

Не вважається `done`:

- "майже готово";
- "має працювати";
- "ймовірно виправлено";
- неперевірений результат;
- результат, при якому перевірки запускались і впали;
- змішані факти й припущення;
- приховані або замовчані обмеження;
- незавершені частини без явного статусу;
- відсутність інформації про невиконані перевірки;
- фінальний звіт, який не відповідає фактичному результату.

## 4. Partial completion / недоведеність

Якщо лишилися частково невирішені або недоведені моменти:

- не позначай задачу як повністю завершену;
- використовуй явний статус `Partial` або `Incomplete` з переліком того, що не закрито;
- явно вкажи їх у `Недоведено / ризики`;
- не підмінюй відсутність доказу впевненим формулюванням;
- не маскуй обмеження під виглядом завершеного результату.
